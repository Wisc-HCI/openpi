Fine-tuning Pi0.5-DROID on CHTC

A step-by-step setup guide for using UW-Madison CHTC, Hugging Face datasets, and the Wisc-HCI DROID/OpenPI pipeline to fine-tune Pi0.5-DROID policies.

CHTC GPU jobs Hugging Face datasets LeRobot format OpenPI Pi0.5-DROID
Use /home/<netid> for code, submit files, and logs. Use /staging/<first-letter>/<netid> for large compressed data, and move large files through transfer.chtc.wisc.edu.

System Roles

This pipeline has three practical locations. Keeping their responsibilities separate makes the setup easier to debug.

Location Purpose Typical files
Robot laptop / local workstation Collect DROID-style RealSense demonstrations and optionally convert them. Raw trajectory.h5, camera MP4s, local converted datasets.
Hugging Face Hub Share converted LeRobot datasets with lab members and CHTC jobs. Private dataset repos such as <org>/realsense_droid.
CHTC Submit HTCondor jobs and run fine-tuning on GPU execute nodes. Submit files, compressed inputs, logs, checkpoints.

Configure Hugging Face Access

Create a Hugging Face user access token from your account settings. Use read permission for downloading private datasets and write permission for uploading datasets or checkpoints.

Authenticate locally
hf auth login
hf auth whoami

If the CHTC Python environment is too old for the latest hf CLI, pass the token as an environment variable instead of relying on interactive login.

Pass token into a job
export HF_TOKEN=hf_your_token_here
HTCondor submit file
getenv = HF_TOKEN

Prepare the Repositories

On CHTC, log into an Access Point and clone the OpenPI branch used for fine-tuning.

Clone OpenPI on CHTC
ssh <netid>@ap2002.chtc.wisc.edu

git clone https://github.com/Wisc-HCI/openpi.git
cd openpi
git checkout pi05-droid-finetune

The DROID repository is used on the robot/control laptop side.

Clone DROID on robot laptop
git clone https://github.com/Wisc-HCI/droid.git
cd droid
git checkout pi05-droid-finetune

Convert RealSense DROID Data

Raw DROID-style demonstrations should contain successful episodes with a trajectory file and camera recordings.

Expected data shape
data/
  success/
    episode_x/
      trajectory.h5
      recordings/
        MP4/
          <camera_id>.mp4

Validate a few episodes before doing the full conversion.

Dry run conversion
uv run examples/droid/convert_realsense_droid_data_to_lerobot.py \
  --data-dir /path/to/droid/data \
  --repo-id <hf-org>/<dataset-name> \
  --dry-run

Then convert the full dataset.

Convert dataset
uv run examples/droid/convert_realsense_droid_data_to_lerobot.py \
  --data-dir /path/to/droid/data \
  --repo-id <hf-org>/<dataset-name> \
  --overwrite

To upload directly after conversion, add --push-to-hub and --private.

Convert and upload
uv run examples/droid/convert_realsense_droid_data_to_lerobot.py \
  --data-dir /path/to/droid/data \
  --repo-id <hf-org>/<dataset-name> \
  --overwrite \
  --push-to-hub \
  --private

Upload and Download Datasets

Use Hugging Face as the lab-facing dataset registry. Always include --repo-type dataset for dataset repos.

Upload a converted dataset
hf upload <hf-org>/<dataset-name> \
  ~/.cache/huggingface/lerobot/<hf-org>/<dataset-name> \
  . \
  --repo-type dataset
Upload a very large dataset
hf upload-large-folder <hf-org>/<dataset-name> \
  ~/.cache/huggingface/lerobot/<hf-org>/<dataset-name> \
  --repo-type dataset
Download a private dataset
hf download <hf-org>/<dataset-name> \
  --repo-type dataset \
  --local-dir ./realsense_droid \
  --token "$HF_TOKEN"

Stage Large Files on CHTC

For CHTC, compress many small files into a few large files before placing them in staging.

Compress inputs
tar -czf realsense_droid.tar.gz realsense_droid/
tar -czf openpi.tar.gz openpi/

Upload large files through the transfer host from your local machine.

Upload to staging
scp realsense_droid.tar.gz \
  <netid>@transfer.chtc.wisc.edu:/staging/<first-letter>/<netid>/
Submit jobs from /home/<netid>, not from /staging. Staging is for large input and output files, not for active job submission directories.

Submit the Fine-tuning Job

A typical submit file requests one large-memory GPU, transfers the compressed dataset, and requires execute nodes with staging access.

pi05_droid_finetune.sub
universe = container
container_image = docker://nvcr.io/nvidia/pytorch:24.05-py3

executable = run_pi05_droid_finetune.sh

log = logs/pi05_droid_$(Cluster).log
output = logs/pi05_droid_$(Cluster).out
error = logs/pi05_droid_$(Cluster).err

request_gpus = 1
gpus_minimum_memory = 80000M
request_cpus = 8
request_memory = 128GB
request_disk = 200GB

getenv = HF_TOKEN

transfer_input_files = file:///staging/<first-letter>/<netid>/realsense_droid.tar.gz, openpi.tar.gz
Requirements = (Target.HasCHTCStaging == true)

queue

The executable script should unpack files, set local caches, install dependencies, and launch training.

run_pi05_droid_finetune.sh
#!/bin/bash
set -euo pipefail

tar -xzf openpi.tar.gz
tar -xzf realsense_droid.tar.gz

cd openpi

export HF_HOME="$PWD/.cache/huggingface"
export OPENPI_DATA_HOME="$PWD/.cache/openpi"
export WANDB_MODE=offline

uv sync

uv run scripts/train.py pi05_droid_finetune \
  --exp-name=realsense_droid \
  --overwrite
Submit from home
mkdir -p ~/logs
condor_submit pi05_droid_finetune.sub

Monitor Training

Use condor_q to check job status, then inspect the corresponding log, output, and error files.

Watch job status and logs
condor_q

tail -f ~/logs/pi05_droid_<cluster_id>.out
tail -f ~/logs/pi05_droid_<cluster_id>.err
tail -f ~/logs/pi05_droid_<cluster_id>.log

If a job is held, ask Condor why it cannot run.

Diagnose held job
condor_q -hold
condor_q -better-analyze <cluster_id>

Retrieve Checkpoints

After training finishes, copy the checkpoint directory back to the lab GPU workstation or policy server.

Copy checkpoints
rsync -avh --progress \
  <netid>@ap2002.chtc.wisc.edu:~/openpi/checkpoints/pi05_droid_finetune/realsense_droid/ \
  ~/Desktop/repo/openpi/checkpoints/pi05_droid_finetune/realsense_droid/

Then serve the fine-tuned policy on the GPU workstation.

Serve fine-tuned policy
uv run scripts/serve_policy.py policy:checkpoint \
  --policy.config=pi05_droid \
  --policy.dir=checkpoints/pi05_droid_finetune/realsense_droid

On the DROID control laptop, point the robot client to the policy server.

Run robot client
python3 scripts/main.py \
  --remote_host=<policy-server-ip> \
  --remote_port=8000 \
  --external_camera="left" \
  --max_timesteps=800

Troubleshooting

Hugging Face 401 or Repository Not Found

This usually means the repo id is wrong, the dataset is private, the token was not passed into the job, or the command forgot --repo-type dataset.

Check token without leaking it
hf auth whoami
echo "$HF_TOKEN" | head -c 8

Job Cannot See Staging

Add a staging requirement to the submit file.

Require staging access
Requirements = (Target.HasCHTCStaging == true)

Boolean CLI Argument Error

For boolean flags, use the CLI's expected flag style.

Correct boolean flag style
--no-wandb-enabled

Avoid this form if the CLI does not accept explicit boolean values:

Likely to fail
--wandb-enabled false

Too Many Small Files

Compress the dataset before staging or transferring it.

Compress dataset
tar -czf realsense_droid.tar.gz realsense_droid/
